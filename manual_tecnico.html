Sistema de Gestão Financeira

Documentação técnica completa para instalação, configuração, manutenção e evolução do produto.

1. Visão Geral

Aplicação web desenvolvida em Python/Flask para controle de contas a pagar e receber, com foco em fluxo de caixa real, dashboard interativo, relatórios analíticos e backup automatizado.

💡 Diferencial: Todos os indicadores (KPIs, gráficos e relatórios) utilizam o campo valor_pago para refletir a realidade financeira efetiva, não apenas projeções.

3. Instalação

# 1. Clone ou extraia o projeto
cd Projeto_Contas_Orcamento

# 2. Crie ambiente virtual
python -m venv venv
venv\Scripts\activate  # Windows
source venv/bin/activate  # Linux/Mac

# 3. Instale dependências
pip install -r requirements.txt

# 4. Configure variáveis de ambiente
cp .env.example .env
# Edite .env com suas credenciais

# 5. Execute em desenvolvimento
python app.py
# Acesse: http://localhost:5000

4. Configuração

Arquivo .env

# Banco de Dados

# Flask

FLASK_ENV=development
FLASK_DEBUG=True

# Uploads & Limites
MAX_CONTENT_LENGTH=16777216  # 16MB

⚠️ Nunca commite o arquivo .env no Git. Utilize .gitignore para excluí-lo do versionamento.

6. Funcionalidades Principais

  • ✅ CRUD Completo: Cadastro, edição, exclusão e busca de contas
  • ✅ Sincronia Inteligente: Selecionar categoria atualiza automaticamente o botão Receita/Despesa
  • ✅ Dashboard KPI: Total, Receitas, Despesas, Saldo Líquido (baseado em valor_pago)
  • ✅ Gráficos Interativos: Categorias, Evolução Mensal, Status, Ranking e Tendência Acumulada com filtros
  • ✅ Relatórios PDF/CSV: Mensal, Anual Detalhado, Por Conta e Anual Receita/Despesa
  • ✅ Backup/Restore: Exportação segura via pg_dump e restauração controlada
  • ✅ Anexos: Upload e visualização de comprovantes em PDF

8. Lógica de Negócio (Destaques)

🔄 Sincronia Categoria ↔ Tipo (Receita/Despesa)

O sistema detecta automaticamente o tipo financeiro baseado no texto da categoria. Se contiver (RECEITA), SALÁRIO ou APOSENTADORIA, o botão +R torna-se verde e o campo oculto é atualizado para R antes do submit.

💰 Cálculo por valor_pago

Todos os indicadores, gráficos e relatórios utilizam valor_pago > 0 como filtro. Isso garante que o dashboard reflita o caixa real, não compromissos futuros.

📈 Tendência Acumulada com Filtros

O gráfico de linha acumula mês a mês respeitando os parâmetros ?categoria=X ou ?conta=Y. A API recalcula a série histórica dinamicamente.

9. Relatórios

  • Mensal: Lista detalhada com status (Pago/Pendente/Atrasado)
  • Anual Detalhado: Agrupamento por conta com totais
  • Por Conta: Histórico completo de um fornecedor/recebedor
  • Anual Receita/Despesa: Tabela de evolução mensal com saldo acumulado e indicadores de desempenho
✅ Os relatórios são gerados via FPDF com formatação brasileira (R$ X.XXX,XX) e codificação Latin-1 compatível.

11. Segurança & Boas Práticas

  • 🔒 SQL Injection: Mitigado via SQLAlchemy ORM (parameterized queries)
  • 🛡️ Path Traversal: Validação de caminhos em uploads e backups
  • 🔑 Secret Key: Gerada aleatoriamente, nunca hardcodeada
  • 📦 Validação de Inputs: Sanitização de valores monetários e datas no backend
  • 🔄 Transações: db.session.rollback() em blocos try/except
